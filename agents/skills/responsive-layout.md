# Skill: 响应式布局
## 技能概述
确保应用在手机优先前提下，在平板和桌面仍保持清晰、可演示、可交互。
## 断点策略
- 默认：移动端（375基准）
- sm: 640+
- md: 768+
- lg: 1024+
- xl: 1280+
## 全局布局规范
1. 页面主体采用居中容器，桌面端保留“手机壳”观感。
2. 底部导航固定，正文必须预留 pb-20。
3. 横向滚动区域必须隐藏滚动条。
建议容器：
```tsx
<main className="mx-auto w-full max-w-md md:max-w-lg lg:max-w-xl min-h-screen bg-white relative">
  {children}
</main>
```
## 页面级规则
### 首页
- 场景入口横向滑动
- 榜单移动端滑动，平板可转网格
- Feed移动端单列，平板可双列
### 详情页
- 封面图16:9
- 必点菜横向滑动
- 底部CTA固定
### AI页
- 消息气泡移动端占比不超过80%
- 输入框固定底部
- 快捷标签可滑动或换行
### 地图页
- 地图区高度优先
- 筛选栏放在地图上方
- 信息卡底部弹出
## 安全区域与触摸
- 使用 env(safe-area-inset-bottom)
- 可点击区域不小于 44x44
- 禁止关键按钮被键盘遮挡
## 验收标准
- [ ] 360-430 宽度无横向溢出
- [ ] iOS安全区域下底部导航不遮挡
- [ ] 所有按钮点击区>=44x44
- [ ] 桌面端为居中展示，不拉满全屏
- [ ] 文字最小12px可读
