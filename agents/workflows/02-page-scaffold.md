# Workflow: 页面脚手架搭建
## 触发条件
完成项目初始化后执行。
## 前置条件
- 应用可运行
- 基础布局组件可用
- 模拟数据可访问
## 分阶段搭建
### Phase 1：首页（最高优先）
- 顶部：高校选择器+搜索框
- AI入口卡片（首屏主CTA）
- 场景入口（横滑）
- 榜单摘要
- 推荐Feed
验收：
- [ ] 首屏结构完整
- [ ] 切校后数据联动
- [ ] 卡片可跳详情
### Phase 2：详情页（/restaurant/[id]）
- 基础信息
- 标签与优惠
- 必点菜
- 评价列表
- 底部导航CTA
### Phase 3：AI页（/ai-pick）
- 聊天区
- 快捷标签
- 推荐卡
- 输入框固定底部
### Phase 4：地图页（/map）
- 筛选栏
- 地图区域
- 点位与信息卡
### Phase 5：榜单页（/rankings）
- 类型Tab
- 排名列表
- 学校过滤
### Phase 6：个人页（/profile）
- 用户信息
- 足迹进度
- 收藏与偏好入口
## 全局要求
1. 保持底部导航一致并高亮当前页。
2. 页面必须支持 360-430 移动端。
3. 页面跳转使用 Next Link 或 router。
4. 数据优先来自 src/backend/api 与 src/backend/data。
## 完成标准
- [ ] 6个页面路由可访问
- [ ] 主链路可跑通
- [ ] 移动端无关键布局问题
