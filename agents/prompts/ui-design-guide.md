# UI Design Guide - UI/UX 设计规范提示词

## 1. 目标与适用范围
本规范用于指导 AI 生成大学城美食地图项目的前端界面。
适用目录：

- src/frontend/app
- src/frontend/components
- src/frontend/styles

设计目标：

1. 决策效率优先：用户在 90 秒内完成一次有效决策。
2. 移动端优先：优先保障 360-430 宽度可用。
3. 风格统一：组件基于 shadcn/ui，避免视觉碎片化。
4. 信息前置：预算、步行时长、同校热度必须高可见。

## 2. 页面级设计原则

1. 一页一主目标

- 首页：触发决策（场景入口或 AI 入口）
- 推荐页：缩短比较路径（给 3 个候选，不给 30 个列表）
- 详情页：促进行动（导航、收藏、分享）

2. 一屏一主 CTA

- 首屏必须能看到主按钮
- 每页主按钮只能有一个视觉主导

3. 先答案，后解释

- 推荐结果先给结论，再给推荐理由
- 避免长段说明压住核心决策信息

## 3. 视觉系统规范

### 3.1 语义色

- brand-primary: #FFD100
- brand-accent: #FF6633
- bg-page: #F7F7F7
- bg-card: #FFFFFF
- text-primary: #1A1A1A
- text-secondary: #666666
- border-default: #EEEEEE
- success: #22C55E
- warning: #F59E0B
- danger: #EF4444

规则：

- 品牌色用于主操作和关键状态，不用于大面积正文。
- 错误信息使用 danger，避免用主色表达错误状态。

### 3.2 字体层级

- Page Title: text-xl font-bold
- Section Title: text-lg font-semibold
- Card Title: text-base font-semibold
- Body: text-sm
- Caption/Meta: text-xs

规则：

- 同一页面不超过 4 个字号层级。
- 辅助信息必须弱于主决策信息。

### 3.3 间距与圆角

- 基础单位：4px
- 页面水平内边距：16px
- 卡片内边距：12px 或 16px
- 卡片圆角：12px
- 组件垂直间距：8/12/16 递进

规则：

- 不允许使用随机 margin 值破坏节奏。
- 底部导航页面统一预留 pb-20。

## 4. 组件规范（shadcn/ui）

### 4.1 必用基础组件

- Button
- Card
- Badge
- Tabs
- Dialog
- Skeleton
- Select
- Input

规则：

- 禁止重复造基础按钮、输入框和卡片。
- 业务组件必须由基础组件组合实现。

### 4.2 餐厅卡片（business）

信息顺序固定：

1. 店名 + 品类
2. 评分 + 同校热度
3. 人均价格 + 步行时长
4. 场景标签（最多 3 个）
5. 推荐理由（1-2 条）
6. 操作按钮（查看详情）

约束：

- 首屏必须显示人均和步行时长。
- 标签超出时折叠为 +N，不允许换行撑爆卡片。

### 4.3 场景入口卡片

- 横向滑动
- 图标 + 短文案（4-8 字）
- 选中态高亮明显

约束：

- 至少保留 5 个固定场景入口。
- 每个入口点击后必须有可见结果反馈。

### 4.4 AI 入口卡片

- 首页首屏可见
- 视觉权重大于普通 feed 区块
- 包含一句行动文案和一个主按钮

## 5. 交互与动效规范

### 5.1 交互反馈

- 所有可点击元素具备 hover/active/focus 态
- 主按钮点击后进入 loading 态并禁用重复触发

### 5.2 动效建议

- 按钮按压：active:scale-95
- 卡片进场：opacity + translateY 轻动效
- 加载状态：Skeleton 优先

约束：

- 单个页面动效时长建议 120-240ms。
- 禁止影响读取内容的持续动画。

## 6. 状态页规范

每个核心页面必须包含：

- loading
- empty
- error

状态文案必须包含下一步动作：

- empty: 放宽条件 / 换个场景
- error: 重试 / 返回首页

## 7. 响应式规范

- 主设计宽度：375
- 可用范围：360-430
- 平板与桌面保持内容居中和宽度上限

硬性约束：

- 不允许横向滚动条
- 不允许按钮超出可视区域
- 不允许底部导航遮挡 CTA

## 8. 可访问性规范

- 文字与背景对比度满足基本可读性
- 交互元素最小点击区域 40x40
- 表单输入有清晰 label 和错误提示

## 9. 输出检查清单

AI 在输出页面代码前必须自检：

1. 是否使用了 shadcn/ui 基础组件
2. 是否包含 loading/empty/error
3. 主 CTA 是否首屏可见
4. 决策核心信息是否首屏可读
5. 360-430 下是否无横向滚动

## 10. 可复用 Prompt 模板

```text
你是本项目前端 UI 设计实现助手。请基于 UI Design Guide 生成页面代码。

输入：
- 页面名称：{pageName}
- 页面目标：{goal}
- 必须字段：{requiredFields}
- 主CTA：{primaryAction}

硬约束：
1) 使用 shadcn/ui 基础组件组合
2) 移动端优先（360-430 可用）
3) 输出 loading/empty/error 三种状态
4) 决策关键信息（价格/步行/热度）首屏可见
5) 代码放在 src/frontend 下对应目录

输出：
- 组件代码
- 状态处理说明
- 自检结果（5条）
```
