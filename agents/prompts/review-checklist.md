# Review Checklist - 自检清单

## 1. 使用说明
本清单用于每次迭代后的统一质检，覆盖：

- 文档一致性
- 前端体验
- 后端接口
- 数据真实性
- 演示可用性

检查优先级：

- Blocker: 必须修复，否则不可演示
- Major: 建议本轮修复，否则影响体验
- Minor: 可在下一轮修复

## 2. Blocker 级检查（必须通过）

### 2.1 主链路闭环

- [ ] 首页 -> 推荐页 -> 详情页 -> 行动（导航/收藏）可完整走通
- [ ] 任意关键路由无 404/空白页
- [ ] 无阻断性交互错误（按钮无响应、崩溃）

### 2.2 数据与接口可用

- [ ] 推荐接口返回结构符合约定（含 items/reasons）
- [ ] 详情接口可返回核心字段（价格、距离、评分）
- [ ] 页面不因单条脏数据崩溃

### 2.3 构建与运行

- [ ] 本地构建通过
- [ ] 类型检查通过（无 TS error）
- [ ] 核心页面可在移动端尺寸正常打开

## 3. Major 级检查（强烈建议通过）

### 3.1 视觉一致性

- [ ] 组件风格统一使用 shadcn/ui 体系
- [ ] 颜色、字号、间距符合 design-system 规范
- [ ] 首屏主 CTA 视觉层级明确

### 3.2 状态与交互

- [ ] 每个核心页面有 loading/empty/error
- [ ] empty/error 有明确下一步动作
- [ ] 可点击元素均有反馈态（hover/active/focus）

### 3.3 移动端适配

- [ ] 360/375/430 宽度均无横向滚动
- [ ] 底部导航不遮挡正文与按钮
- [ ] 文本不截断关键信息

### 3.4 内容质量

- [ ] 推荐理由与输入条件一致
- [ ] 评价文案有学生语境且不模板化
- [ ] 价格、步行时长、场景标签逻辑一致

## 4. Minor 级检查（可排期）

- [ ] 动效细节统一（时长、缓动曲线）
- [ ] 文案语气进一步统一
- [ ] 组件拆分进一步优化
- [ ] 日志与埋点字段补全

## 5. 文档一致性检查

- [ ] docs/product/feature-list.md 与当前实现一致
- [ ] docs/tech/api-design.md 与接口实现一致
- [ ] docs/README.md 阅读路径与当前文件一致
- [ ] 变更已记录到 deliverables/ai-usage-log.md（如有重大调整）

## 6. 代码检查命令建议

```text
1) npm run type-check
2) npm run lint
3) npm run build
4) npm run dev（手动走主链路）
```

## 7. 缺陷记录模板

```text
[Issue]
级别: Blocker | Major | Minor
模块: frontend | backend | docs | data
位置: 文件路径 + 页面/接口
现象: 简要描述
复现步骤: 1,2,3
期望结果: 简要描述
修复建议: 简要描述
负责人: 
状态: todo | doing | done
```

## 9. 可复用 Prompt 模板

```text
请作为本项目的质量审查助手，严格按照 Review Checklist 执行检查。

输入：
- 变更文件列表
- 当前可运行页面与接口

输出：
1) Blocker/Major/Minor 三段式问题清单
2) 每个问题给出最小修复建议
3) 给出“是否可演示”的结论
4) 生成自检报告草稿
```
