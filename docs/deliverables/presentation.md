# 演示话术（Presentation）

## 1. 开场（30秒）

本项目是「大学城美食地图」，目标不是做一个餐厅列表，而是解决大学生每天高频的“吃什么”决策问题。
我们的核心判断是：现有平台信息很多，但学生决策效率不足。
因此，V1聚焦“AI推荐 + 场景化入口 + 高校锚点地图”，让用户在90秒内完成一次可行动决策。

## 2. 目标与成功标准（1分钟）

### 2.1 目标

- 目标用户：大学城在校学生（18-26岁）
- 核心问题：决策疲劳、信息分散、学生场景不匹配
- 核心目标：缩短“打开应用到行动”的路径

### 2.2 成功标准（Demo）

- P0主链路可跑通：首页 -> AI帮我选 -> 餐厅详情 -> 导航/收藏
- 推荐结果可解释：预算、距离、同校热度至少2项可见
- 异常有兜底：无结果时可一键放宽条件

## 3. 方案说明（2-3分钟）

### 3.1 方案1：场景化入口替代传统分类

- 传统平台按菜系分类，学生实际按场景决策。
- 本产品用“一个人吃/聚餐/约会/夜宵/穷鬼快乐餐”作为一级入口。
- 价值：降低筛选成本，直接进入可用结果。

### 3.2 方案2：AI帮我选（核心差异）

- 用户可直接输入约束条件：预算、人数、距离、口味。
- 系统返回Top3候选，并附推荐理由。
- 价值：把“搜索与比较”变为“一次表达需求”。

### 3.3 方案3：学生信任机制

- 使用同校热度和学生评价摘要增强可信度。
- 将“步行分钟数”前置，强化可达性判断。
- 价值：减少“看了很多但不敢去”的决策犹豫。

## 4. 指标设计（1分钟）

北极星指标：决策转化率（进入应用后触发导航/收藏/分享的比例）。

核心指标：

- 激活率：点击场景入口或AI入口的用户占比
- 推荐采纳率：点击推荐详情的用户占比
- 行动转化率：导航/收藏触发率
- 反馈率：喜欢/不喜欢的反馈提交率

## 5. 风险与对策（1分钟）

### 5.1 风险

- 数据覆盖不足导致推荐结果稀疏
- 推荐质量不稳定影响信任
- 方案看起来像“简化版点评”

### 5.2 对策

- 先确保每校核心商圈基础数据覆盖
- 用规则兜底保证推荐可解释
- 在演示中重点突出“场景入口 + AI决策助手”差异

## 6. 迭代规划（30秒）

- V1.5：完善反馈学习闭环、增加更多榜单主题
- V2.0：接入真实地图API与定位能力
- V3.0：探索与本地生活平台交易能力打通

## 7. 结束语（15秒）

这个项目想证明的不是“我能做一个页面”，而是“我能定义一个更高效的学生决策产品”。
AI是实现工具，产品判断力和优先级管理是核心竞争力。
