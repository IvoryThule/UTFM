# AI 工具使用记录（AI Usage Log）

## 1. 记录目的

- 展示AI Coding协作过程，而非只展示最终结果。
- 体现候选人在“问题定义 -> Prompt设计 -> 迭代修正 -> 交付沉淀”上的能力。

## 2. 使用工具清单

- Claude / GPT：产品文档生成、方案迭代、结构化表达优化
- Github Copilot：目录重构、文档落盘、工程化推进
- Cursor：后续代码实现与debug

## 3. 模块化使用记录

### 3.1 文档模块

目标：先完成“产品定义”再进入实现，确保方向正确。

关键Prompt示例：

```text
请基于“大学城学生吃什么决策困难”场景，输出一个可用于AI产品经理测评的PRD，重点强调：
1) 用户痛点与场景细分
2) 场景化信息架构
3) AI推荐差异化
4) Demo与规划功能边界
```

迭代动作：

- 将泛化“用户画像”拆分成独立文档，补充可验证假设与指标。
- 将“流程描述”补充为状态机与异常兜底。
- 将“功能描述”补充为P0/P1/P2与验收标准。

产出：

- PRD、用户调研、用户流程、信息架构、功能清单、竞品分析、演示话术。

### 3.2 前端模块（规划中）

目标：快速搭建可演示主链路页面。

关键Prompt示例：

```text
请用 Next.js + TypeScript 搭建移动优先页面：
首页（场景入口+AI入口）/ 地图页 / AI推荐页 / 详情页。
要求：卡片信息包含步行时间、同校热度、预算标签。
```

预期迭代动作：

- 先搭静态页面骨架，再接Mock数据。
- 保证“首页 -> 推荐 -> 详情 -> 行动”路径一镜到底。
- 增加轻交互（换一批、收藏反馈）提升演示质感。

### 3.3 后端模块（规划中）

目标：用轻量Mock API支撑演示，不追求复杂后端。

关键Prompt示例：

```text
请设计3个接口：
GET /api/restaurants
POST /api/recommend
POST /api/summary
并提供字段定义和Mock返回示例。
```

预期迭代动作：

- 先规则化推荐（预算、距离、场景）确保稳定可解释。
- 再补充摘要能力（评价提炼）增强AI感知。

## 4. 关键协作策略

1. 先定义问题，再让AI写内容：避免“看起来完整但方向错误”。
2. 一次只优化一个变量：例如先改结构，再改文案。
3. 每次迭代都落盘：形成可审计的交付过程。
4. 先可演示，再完美：优先保证主链路可跑通。

## 5. 结果复盘

- 成功点：文档生产效率高，结构化表达显著提升。
- 风险点：如果Prompt不收敛，容易出现冗长与同质化内容。
- 优化点：后续增加模板化Prompt库，减少重复沟通成本。
