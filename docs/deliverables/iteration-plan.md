# 迭代规划（Iteration Plan）

## 1. 规划原则

- 先验证核心价值：先证明“决策效率提升”成立。
- 先做主链路闭环：发现 -> 推荐 -> 行动 -> 反馈。
- 指标驱动迭代：每个版本绑定清晰目标与衡量指标。

## 2. 版本路线图

### V1.0（当前Demo版本）

目标：完成最小可演示闭环，突出AI产品差异。

功能范围：

- P0主链路：首页、AI推荐、餐厅详情、行动按钮
- 基础榜单与场景入口
- Mock数据与可解释推荐理由

核心指标目标：

- 激活率 >= 55%
- 推荐采纳率 >= 35%
- 决策转化率 >= 25%

### V1.5（体验增强版本）

目标：提升推荐信任与复访意愿。

功能范围：

- 推荐反馈闭环（喜欢/不喜欢）
- 更多场景模板（考前续命、月末省钱等）
- 榜单动态更新机制

核心指标目标：

- 推荐采纳率提升到 >= 42%
- 次日留存 >= 18%
- 反馈提交率 >= 20%

### V2.0（地图与本地化升级）

目标：增强空间决策能力与真实可达性。

功能范围：

- 接入真实地图API（高德/Mapbox）
- 实时营业状态与路线时间
- 多高校区域热力分布

核心指标目标：

- 地图页使用占比 >= 30%
- 导航点击率提升到 >= 32%
- 平均决策时长下降到 <= 80秒

### V2.5（社交增长版本）

目标：提升自然传播与多人决策效率。

功能范围：

- 二选一PK与分享投票
- 宿舍/社团聚餐协同决策卡片
- 美食足迹与轻量成就体系

核心指标目标：

- 分享率 >= 12%
- 社交链路转化率 >= 8%
- 7日留存 >= 15%

### V3.0（商业化探索版本）

目标：从“决策工具”升级为“校园本地生活入口”。

功能范围：

- 学生优惠券聚合与分发
- 商户推广位与主题活动合作
- 与本地生活交易能力打通（视合作条件）

核心指标目标：

- 商户合作数持续增长
- 优惠领取转化率 >= 10%
- 商业化模块不损害用户决策效率指标

## 3. 指标体系与看板建议

北极星指标：决策转化率（触发导航/收藏/分享的用户占比）。

辅助指标：

- 用户侧：DAU、次日/7日留存、人均决策次数
- 推荐侧：AI提交率、推荐采纳率、推荐满意度
- 内容侧：榜单点击率、详情停留时长
- 商业侧：优惠点击率、合作商户数（V3阶段）

## 4. 风险与里程碑控制

| 风险 | 影响 | 控制策略 |
|---|---|---|
| 数据更新慢 | 推荐失真、信任下降 | 先做重点商圈高频更新机制 |
| 指标好看但价值不实 | 误判方向 | 优先看“决策转化率”而非单纯停留时长 |
| 功能膨胀 | 影响交付节奏 | 严格按P0/P1/P2执行，版本内冻结范围 |

## 5. 阶段性验收标准

1. 每个版本必须有“可演示主链路”。
2. 每个版本必须有“至少1个核心指标达成或显著改善”。
3. 每次新增功能必须说明对北极星指标的贡献路径。
