# 产品流程设计（Product Flow）

## 1. 文档目标
定义「大学城美食地图」V1核心业务流程，明确页面跳转关系、状态流转与异常处理，作为前后端实现蓝图。

## 2. 核心主链路（发现 -> 推荐 -> 决策 -> 行动）

```text
启动应用
  -> 首页（学校已选）
    -> 入口A：场景卡片
    -> 入口B：AI帮我选
    -> 入口C：榜单/推荐Feed
      -> 候选餐厅列表
        -> 餐厅详情
          -> 行动按钮（导航 / 收藏 / 分享）
            -> 行动完成（闭环）
              -> 反馈（喜欢/不喜欢）
                -> 偏好更新（用于下一次推荐）
```

## 3. 页面级流程

### 3.1 首页流程

1. 用户进入首页，系统默认定位到上次选择学校（无历史则给默认学校）。
2. 用户可在顶部切换学校，页面餐厅数据按学校上下文刷新。
3. 首页提供三类入口：

- 场景入口：一个人吃/情侣约会/宿舍聚餐/深夜食堂/穷鬼快乐餐
- AI入口：输入自然语言需求
- 内容入口：热门榜单与推荐Feed

4. 任一入口均流向候选餐厅结果集（列表或卡片）。

### 3.2 AI推荐流程

1. 用户在AI页输入约束：预算、人数、口味、距离、场景。
2. 推荐引擎执行筛选与排序（规则+语义匹配）。
3. 返回3个候选项并展示推荐理由：

- 预算匹配
- 步行时长
- 同校热度
- 场景契合标签

4. 用户动作分流：

- 点详情：进入餐厅详情页
- 换一批：保留约束重新出结果
- 二选一PK：进入对比模式

### 3.3 餐厅详情流程

1. 展示基础信息：营业时间、人均、地址、电话、学生优惠。
2. 展示学生视角内容：标签、必点菜、同校评价摘要。
3. 提供关键行动按钮：

- 导航到店（主按钮）
- 收藏（次按钮）
- 分享给好友（次按钮）

4. 完成行动后返回来源页并保留上下文（滚动位置与筛选条件）。

## 4. 状态机视角（简化）

### 4.1 推荐状态

- idle：未输入需求
- loading：生成推荐中
- success：返回候选餐厅
- empty：无匹配结果
- error：请求失败

状态流转：

- idle -> loading -> success
- idle -> loading -> empty
- idle -> loading -> error
- empty/error -> loading（用户重试）

### 4.2 用户决策状态

- browsing：浏览候选项
- comparing：对比候选项
- deciding：进入详情确认
- acting：执行导航/收藏/分享
- completed：完成一次有效决策

## 5. 异常与兜底流程

| 异常场景 | 触发条件 | 兜底策略 |
|---|---|---|
| 推荐无结果 | 条件过严（预算过低+距离过近） | 提示放宽条件并给“一键放宽”按钮 |
| 数据加载失败 | 网络异常或接口错误 | 显示重试按钮与本地缓存推荐 |
| 学校切换后无数据 | 新学校数据覆盖不足 | 回退到热门榜单+提示“数据建设中” |
| 夜间可用店少 | 时段过滤后结果不足 | 自动扩展半径并标注“更远可选” |

## 6. 关键决策节点（评测展示重点）

1. 节点N1：用户是否进入场景入口或AI入口（激活）
2. 节点N2：用户是否点击推荐结果（采纳意愿）
3. 节点N3：用户是否触发导航/收藏（决策转化）
4. 节点N4：用户是否提交反馈（学习闭环）

## 7. API映射建议（与目录结构对齐）

- GET /api/restaurants
  - 用途：首页列表、地图点位、详情基础信息
- POST /api/recommend
  - 用途：AI帮我选、换一批、二选一候选生成
- POST /api/summary
  - 用途：评价摘要、推荐理由生成

## 8. 与前端页面映射

- src/frontend/app/page.tsx：首页发现与场景入口
- src/frontend/app/ai-pick/page.tsx：AI推荐与对比
- src/frontend/app/map/page.tsx：学校锚点地图浏览
- src/frontend/components/business/restaurantCard.tsx：候选餐厅卡片
- src/frontend/components/business/mapView.tsx：地图渲染与筛选控件
- src/frontend/lib/api.ts：统一请求层

## 8.1 与后端接口映射

- src/backend/api/restaurants.ts：餐厅列表与筛选接口
- src/backend/api/recommend.ts：AI推荐接口
- src/backend/api/summary.ts：评价摘要与推荐理由接口

## 9. 成功标准（V1 Demo）

- 主链路可在3步内完成一次决策行动。
- 任一入口到“导航/收藏”平均不超过90秒。
- 异常场景均有明确提示与可恢复路径。
