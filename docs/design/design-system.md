# 设计系统（Design System）

## 1. 设计目标

- 支持“高频、快速、移动优先”的学生决策场景。
- 在有限开发时间内保持视觉统一与可扩展。
- 让面试演示可清晰说明“为什么这样设计”。
- 基础组件风格与交互基线统一采用 shadcn/ui。

## 2. 设计原则（面试可讲清）

1. 先答案后细节：优先展示可行动信息（去哪吃、几分钟、多少钱）。
2. 先场景后品类：围绕真实决策路径组织界面。
3. 可解释优先：推荐必须给出理由，不做黑盒结果。
4. 单手可操作：主要点击区集中在拇指热区。
5. 一屏一重点：每个页面只强调一个核心动作。

## 3. 视觉基础

### 3.1 色彩（建议值）

- 主色：`#FFC300`（决策主按钮、关键高亮）
- 辅助色：`#FF7A00`（强调状态、活动标签）
- 成功色：`#12B76A`
- 警示色：`#F79009`
- 文本主色：`#1D2939`
- 次级文本：`#667085`
- 背景色：`#F8FAFC`
- 卡片底色：`#FFFFFF`

### 3.2 字体与层级

- 字体：系统无衬线（移动端可读优先）
- 标题：20/24，SemiBold
- 小标题：16/22，Medium
- 正文：14/20，Regular
- 辅助信息：12/18，Regular

### 3.3 间距与圆角

- 基础间距：4px网格（4/8/12/16/24）
- 卡片圆角：12px
- 主按钮圆角：10px
- 页面安全边距：16px

## 4. 信息优先级规则

餐厅卡片信息排序：

1. 名称 + 场景标签
2. 步行时长 + 人均价格
3. 评分 + 同校热度
4. 推荐理由摘要

说明：将用户最关心的决策因子前置，减少来回滑动与对比成本。

## 5. 动效规则（轻量）

- 卡片出现：120-180ms淡入上移
- Tab切换：150ms
- 推荐加载：骨架屏 + 文案提示，不用复杂动画
- 交互反馈：按钮按压态必须可见

## 6. 可用性规则

- 主点击区最小44x44
- 关键按钮文案使用动词（如“帮我选”“去导航”）
- 空状态提供可执行建议（如“放宽预算再试试”）
- 错误状态可恢复（重试按钮+兜底推荐）

## 7. 无障碍与适配

- 文本与背景对比度满足基本可读标准
- 不依赖纯颜色表达状态，辅以图标或文案
- 360px宽度下保证主链路无横向滚动

## 8. 输出约束

- 视觉一致性优先于花哨效果
- 新增组件必须复用既有token（颜色/间距/圆角）
- 演示版本避免引入重交互导致性能波动
- 基础交互组件优先使用 shadcn/ui（Button、Card、Dialog、Tabs、Input）。
