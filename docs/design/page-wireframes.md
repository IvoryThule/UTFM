# 页面线框说明（Page Wireframes）

## 1. 文档目标
用文字线框方式说明核心页面的信息布局与关键交互，便于在无高保真稿时快速对齐实现。

## 2. 首页（src/frontend/app/page.tsx）

## 布局结构

1. 顶部区：学校切换 + 搜索入口
2. 核心区：AI入口Banner（主CTA）
3. 场景区：横向场景卡片
4. 内容区：榜单模块 + 推荐Feed
5. 底部导航：首页/地图/AI/我的

## 交互重点

- 切换学校：全页面数据刷新
- 点击场景：进入过滤后的结果集
- 点击AI入口：跳转AI推荐页

## 3. AI推荐页（src/frontend/app/ai-pick/page.tsx）

## 布局结构

1. 输入区：自然语言输入框 + 快捷标签
2. 操作区：提交按钮/换一批按钮
3. 结果区：Top3推荐卡片
4. 说明区：推荐理由与命中条件

## 交互重点

- 提交后进入loading态
- 返回结果后可查看详情或二选一对比
- 无结果时给出放宽条件建议

## 4. 地图页（src/frontend/app/map/page.tsx）

## 布局结构

1. 顶部区：学校切换 + 筛选条
2. 主体区：地图点位
3. 底部浮层：当前可见区域餐厅列表

## 交互重点

- 点位与列表双向联动
- 筛选变化后结果即时更新

## 5. 餐厅详情页（src/frontend/app/restaurant/[id]/page.tsx）

## 布局结构

1. 头图与基础信息（名称、评分、人均、营业时间）
2. 学生标签与评价摘要
3. 必点菜推荐
4. 行动底栏（导航/收藏/分享）

## 交互重点

- 首屏可见“人均+步行时长+导航按钮”
- 行动按钮固定底部，便于单手操作

## 6. 榜单页（src/frontend/app/rankings/page.tsx）

## 布局结构

1. 榜单Tab：热门/低价/新店
2. 榜单列表：排序名次 + 关键信息

## 交互重点

- 榜单切换不丢失滚动与筛选上下文

## 7. 个人页（src/frontend/app/profile/page.tsx）

## 布局结构

1. 偏好设置（口味、预算、距离）
2. 收藏分组（想吃/吃过/避雷）
3. 最近决策记录

## 交互重点

- 偏好更新后在推荐页可感知变化

## 8. 统一页面规则

1. 每页只保留一个主CTA。
2. 关键决策信息在首屏展示。
3. 空态和错态必须有下一步动作。
4. 返回上级页面保留上下文状态。
